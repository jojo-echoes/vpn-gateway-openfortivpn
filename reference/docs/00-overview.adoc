== 專案概述

本文件系列記錄了一個完整的企業級 VPN 自動化解決方案的開發過程，整合了以下技術：

* **OpenPGP (GPG 2.4.4)**: 提供非對稱加密的密碼管理
* **YubiKey 5C NFC**: 硬體安全金鑰，儲存私鑰並提供雙因素認證
* **pass (Unix Password Manager)**: 基於 GPG 的密碼管理工具
* **openfortivpn**: FortiGate SSL VPN 客戶端
* **Bash 自動化腳本**: 智能偵測 YubiKey 並提供自動/手動雙模式

=== 核心特性

==== 1. 硬體級安全
* 私鑰永不離開 YubiKey（無法導出）
* RSA 4096 位元金鑰
* 雙 YubiKey 冗餘備份
* PIN 保護（3 次錯誤後鎖定）

==== 2. 智能自動化
* 自動偵測 YubiKey 存在
* YubiKey 模式：全自動背景執行
* 手動模式：screen 會話互動輸入
* 無縫切換，無需修改設定

==== 3. 企業級功能
* Server-pushed routes（由 FortiGate 管理路由）
* Split tunneling（保持原有網路存取）
* PID 追蹤（可靠的連接/中斷控制）
* Trap 清理機制（確保臨時檔案清理）

=== 開發歷程重點

==== 關鍵挑戰與解決方案

1. **VPN 路由問題**
   * 問題：手動管理路由導致維護困難
   * 解決：啟用 `set-routes = 1` 接受 server-pushed routes

2. **YubiKey 識別問題**
   * 問題：VMware USB 2.0 無法可靠識別智能卡
   * 解決：升級到 USB 3.1 控制器 + udev 規則

3. **GPG 金鑰產生逾時**
   * 問題：預設 gpg-agent timeout 為 0
   * 解決：設定 `pinentry-timeout 120` 並理解現代 Linux entropy

4. **背景執行密碼提示**
   * 問題：nohup/背景執行無法輸入密碼
   * 解決：(a) YubiKey 模式使用 pipe；(b) 手動模式使用 screen

5. **sudo 權限問題**
   * 問題：背景執行時 sudo 要求密碼
   * 解決：使用 `sudo -v` 預先驗證

6. **臨時檔案清理**
   * 問題：異常退出時臨時檔案殘留
   * 解決：實作 trap 清理機制

=== 安全考量

==== 威脅模型

本系統防護以下威脅：

* ✓ 密碼明文儲存在檔案系統
* ✓ 密碼在記憶體中明文存在時間過長
* ✓ 私鑰被惡意程式竊取
* ✓ 單一金鑰遺失或損壞
* ✓ 腳本歷史記錄洩漏密碼

==== 安全邊界

本系統**無法**防護：

* ✗ Root 權限的惡意程式（可攔截記憶體）
* ✗ 鍵盤記錄器（可攔截 PIN 輸入）
* ✗ 物理存取+知道 PIN 的攻擊者
* ✗ YubiKey 本身的韌體漏洞
